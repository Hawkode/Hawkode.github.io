Work Project

Network Cabling & Port Mapping

This project focused on improving physical network visibility by tracing network cables from server room patch panels to wall sockets, matching patch panel ports with office wall ports, applying clear labels, and documenting port-to-location records.

The goal was to create a reliable reference for troubleshooting, hardware setup, office changes, asset tracking, and day-to-day infrastructure work. This helped reduce uncertainty when identifying which network port served a desk, device, office area, or endpoint.

Privacy note: Real company names, site names, floor plans, port numbers, switch details, desk numbers, screenshots, and internal references have been removed or recreated in a sanitised format.

Problem

Why This Project Was Needed

In office environments, physical network documentation can become outdated over time. Wall sockets may not clearly match patch panel ports, labels can be missing or inconsistent, and desk moves or equipment changes can make it harder to know which connection belongs to which location.

Before

  • Patch panel ports were harder to match to wall sockets quickly.
  • Some physical network paths needed clearer documentation.
  • Desk locations and port references were not always easy to confirm.
  • Troubleshooting could take longer when port records were unclear.
  • Hardware setup and office changes required extra manual checking.

Target State

  • Clear mapping between patch panel ports and wall sockets.
  • Consistent labels for ports, desks, and locations.
  • Reliable port-to-location records.
  • Faster identification of active network points.
  • Better physical network visibility across the site.

Approach

Port Mapping Workflow

The project followed a structured workflow to identify, verify, label, and document network connections in a repeatable way.

Mapping Flow

Patch Panel • Wall Socket • Desk Location • Documentation

Patch panel port
        ↓
Cable trace or connectivity check
        ↓
Wall socket identification
        ↓
Desk or office location confirmed
        ↓
Label applied or checked
        ↓
Port-to-location record updated
        ↓
Used for troubleshooting, setup, moves, and asset visibility

This made the physical network easier to understand and reduced the time needed to locate the correct connection during troubleshooting or equipment setup.

Tracing

Patch Panel to Wall Socket Tracing

Cables were traced from the server room patch panels to office wall sockets. The aim was to confirm which patch panel port served which physical location.

Tasks Completed

  • Reviewed server room patch panel layout.
  • Identified ports that needed clearer records.
  • Matched patch panel ports with wall sockets.
  • Checked desk or area location for each socket.
  • Updated records where mappings were confirmed.

Information Captured

  • Patch panel reference
  • Patch panel port number
  • Wall socket number
  • Desk or office area
  • Connection status
  • Notes for future checks

Connectivity

Connectivity Checks

Connectivity checks were used to confirm that documented ports were accurate and usable. This helped verify whether a wall socket, patch panel port, or desk connection was active and correctly identified.

  • Checked whether wall sockets matched the expected patch panel ports.
  • Confirmed active network points where required.
  • Reviewed connections that did not match existing records.
  • Updated documentation after confirmed checks.
  • Flagged unclear or unused ports for later review.

Operational Use

How The Records Supported IT Work

Troubleshooting: when a user or device had a network issue, the correct wall socket and patch panel path could be identified faster.

Hardware setup: new desks, laptops, desktops, phones, or other networked devices could be connected using clearer port references.

Office changes: desk moves and area changes were easier to plan because physical network points were documented.

Asset tracking: network port records could be linked with device locations, desk references, and asset inventory information.

Site visibility: the server room, patch panels, wall sockets, desks, and office areas became easier to understand as one connected system.

Quality Control

Checks and Validation

After mappings were recorded, the information was reviewed for accuracy and consistency.

  • Checked that patch panel references matched wall socket records.
  • Reviewed labels for clarity and consistency.
  • Confirmed desk or area locations were recorded correctly.
  • Updated unclear or outdated records.
  • Checked records against known hardware locations where possible.
  • Kept notes for ports that needed future review.

Reflection

What This Project Demonstrates

This project demonstrates practical networking and infrastructure awareness. It shows the importance of understanding not only logical network configuration, but also the physical path between the server room, patch panels, wall sockets, desks, and user devices.

Good physical documentation supports faster troubleshooting, better hardware planning, cleaner asset records, and improved visibility across a site. This is especially valuable when working with infrastructure, endpoint management, and operational security because accurate location and connectivity records help teams understand what is connected, where it is located, and how it is being used.
