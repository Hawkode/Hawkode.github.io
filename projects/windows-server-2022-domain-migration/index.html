Work Project

Windows Server 2022 Domain Controller Migration

This project involved building a new Windows Server 2022 domain controller and migrating core infrastructure services, user accounts, Windows 11 Pro laptops, shared data, and identity synchronisation from an older environment.

The new environment included Active Directory Domain Services, DNS, DHCP, Microsoft Entra ID sync, Robocopy data migration, user migration and attribute matching with ADManager Plus, and PowerShell-based domain join automation deployed through ManageEngine.

Migration Plan

Migration Workflow

1. Planning: reviewed the existing environment, user accounts, devices, shared data, and infrastructure services that needed to move.

2. Server Build: installed Windows Server 2022, added required roles, and promoted the server as a domain controller.

3. Network Services: configured DNS and DHCP so clients could resolve domain resources and receive correct network settings.

4. Data Migration: used Robocopy to migrate shared data while preserving structure and permissions where required.

5. User Migration: migrated 50+ user accounts, matched attributes, and completed bulk updates using ADManager Plus.

6. Endpoint Migration: used PowerShell and ManageEngine to automate Windows 11 Pro laptop domain joins.

7. Validation: checked sign-in, DNS, DHCP, shared folders, group membership, Entra ID sync, and event logs.

Endpoint Migration

Windows 11 Pro Domain Join Automation

A PowerShell script was created to join Windows 11 Pro laptops to the new domain. The script was uploaded into ManageEngine so the domain join process could be deployed more consistently across devices instead of joining every laptop manually.

Sanitised PowerShell Example

PowerShell • Domain Join • ManageEngine • Windows 11 Pro

$DomainName = "example.local"
$OUPath = "OU=Laptops,DC=example,DC=local"

Add-Computer -DomainName $DomainName -OUPath $OUPath -Restart
  • Created a PowerShell domain-join script.
  • Uploaded the script into ManageEngine.
  • Joined Windows 11 Pro laptops to the new domain.
  • Confirmed laptops appeared in Active Directory.
  • Checked user sign-in after domain join.
  • Verified DNS and DHCP settings after migration.
  • Troubleshot failed joins where required.
