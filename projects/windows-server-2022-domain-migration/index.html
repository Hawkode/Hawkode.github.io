Work Project

Windows Server 2022 Domain Controller Migration

This project involved building a new Windows Server 2022 domain controller and migrating core infrastructure services, user accounts, Windows 11 Pro laptops, shared data, and identity synchronisation from an older environment.

The new environment included Active Directory Domain Services, DNS, DHCP, Active Directory replication checks, Microsoft Entra ID sync, Robocopy data migration, user account migration and attribute matching with ADManager Plus, and PowerShell-based domain join automation deployed through ManageEngine.

This page describes the project at a high level without exposing internal company details, real domain names, server names, IP addresses, usernames, or file paths.

Migration Plan

Migration Workflow

1. Discovery: reviewed the existing environment, user accounts, Windows 11 Pro laptops, shared data, network services, and identity requirements.

2. Server Build: installed Windows Server 2022, added required roles, and promoted the server as a new domain controller.

3. Directory Design: created a cleaner Active Directory structure for users, departments, computers, groups, disabled accounts, and service accounts.

4. Replication Checks: checked Active Directory replication health so directory changes, DNS records, and domain controller information were consistent before continuing the migration.

5. Network Services: configured DNS and DHCP so client devices could resolve domain resources and receive the correct network settings.

6. User Account Migration: migrated 50+ user accounts, matched attributes, reviewed group membership, and completed bulk updates using ADManager Plus.

7. Data Migration: used Robocopy to migrate shared data, review logs, check folder structure, and validate access after migration.

8. Endpoint Migration: created a PowerShell domain join script and deployed it through ManageEngine to join Windows 11 Pro laptops to the new domain.

9. Hybrid Identity: synced on-premises Active Directory accounts with Microsoft Entra ID and checked cloud identity records after migration.

10. Documentation: documented the migration approach, key checks, commands, validation steps, and post-migration notes for internal use.

Replication

Active Directory Replication and Sync Validation

Replication checks were included because directory changes must be consistent across domain controllers before user migration, endpoint migration, and cloud identity synchronisation can be trusted. If replication fails, user accounts, computer objects, DNS records, group membership, and policy-related data can become inconsistent.

During the migration, replication health was checked around key changes such as domain controller promotion, OU creation, user account migration, DNS updates, and Windows 11 Pro domain joins.

Replication Areas Checked

  • Domain controller replication status
  • DNS replication consistency
  • SYSVOL and NETLOGON availability
  • User and computer object visibility
  • Group membership consistency
  • Event Viewer replication errors

Why It Mattered

  • Prevented stale directory information.
  • Reduced migration risk before joining laptops.
  • Confirmed account changes were visible across the domain.
  • Improved confidence before Microsoft Entra ID sync checks.
  • Made post-migration validation more reliable.

Example Sanitised Replication Checks

Replication • DCDIAG • REPADMIN • DNS • SYSVOL

repadmin /replsummary
repadmin /showrepl
dcdiag /test:replications
dcdiag /test:dns
  • Checked replication summary for failures or delays.
  • Reviewed domain controller health using diagnostic checks.
  • Confirmed DNS records were available after changes.
  • Checked that user and computer objects appeared correctly.
  • Reviewed Event Viewer for directory service and replication warnings.

Endpoint Automation

PowerShell Domain Join Through ManageEngine

A PowerShell script was created to join Windows 11 Pro laptops to the new domain. The script was uploaded into ManageEngine so the domain join process could be deployed consistently across devices instead of joining every laptop manually.

Sanitised PowerShell Example

PowerShell • Domain Join • ManageEngine • Windows 11 Pro

$DomainName = "example.local"
$OUPath = "OU=Laptops,OU=Computers,OU=Company,DC=example,DC=local"

Add-Computer -DomainName $DomainName -OUPath $OUPath -Restart
  • Created a PowerShell domain join script.
  • Configured the target OU path for laptop placement.
  • Uploaded the script into ManageEngine.
  • Joined Windows 11 Pro laptops to the new domain.
  • Confirmed laptop objects appeared in Active Directory.
  • Verified DNS and DHCP settings after migration.
  • Reviewed failed joins and corrected issues.

Hybrid Identity

Microsoft Entra ID Sync

The on-premises Active Directory environment was synced with Microsoft Entra ID to enable hybrid identity. This allowed migrated user accounts to appear in the cloud directory after the on-premises migration.

Because cloud identity depended on on-premises directory data, replication and sync status were checked before validating Microsoft Entra ID records.
