Guide

Command Line Basics for IT, Networking and Security

The command line is one of the most useful skills to learn in IT. It helps with troubleshooting, file management, networking, server administration, scripting, and security labs.

This guide covers basic Linux and macOS terminal commands using simple examples. The goal is to understand what each command does and why it matters.

Getting Started

What is the command line?

The command line is a text-based way to interact with a computer. Instead of clicking through menus, you type commands to move around folders, create files, check system information, run tools, and troubleshoot problems.

In Linux and macOS, the command line is usually accessed through an application called Terminal. 

hawkode@linux:~$

This is called the prompt. It shows that the terminal is ready for a command.

Navigation

Checking where you are: pwd

The pwd command means “print working directory”. It shows your current location in the file system. 

hawkode@linux:~$ pwd
/home/hawkode

This is useful because it tells you exactly where you are before creating, moving, or deleting files.

Files and Folders

Listing files and folders: ls

The ls command lists the files and folders in your current directory. 

hawkode@linux:~$ ls
Desktop  Documents  Downloads  Projects

You can also use ls -la to show hidden files, permissions, owners, file sizes, and modification dates. 

hawkode@linux:~$ ls -la
drwxr-xr-x  5 hawkode hawkode 4096 Jan 10 10:20 .
drwxr-xr-x  3 root    root    4096 Jan 10 09:50 ..
drwxr-xr-x  2 hawkode hawkode 4096 Jan 10 10:10 Documents
drwxr-xr-x  2 hawkode hawkode 4096 Jan 10 10:15 Projects

Navigation

Moving around: cd

The cd command means “change directory”. It lets you move between folders. 

hawkode@linux:~$ cd Documents
hawkode@linux:~/Documents$

To move back one level, use: 

hawkode@linux:~/Documents$ cd ..
hawkode@linux:~$

To return to your home folder, use: 

hawkode@linux:~/Documents$ cd ~
hawkode@linux:~$

Creating Items

Creating folders and files

Use mkdir to create a folder. 

hawkode@linux:~$ mkdir labs

Use touch to create an empty file. 

hawkode@linux:~$ touch notes.txt

These commands are useful when organising lab notes, scripts, logs, and project files.

Reading Files

Viewing file contents: cat

The cat command prints the contents of a file in the terminal. 

hawkode@linux:~$ cat notes.txt
Nmap scan results
Open ports: 22, 80, 443

This is helpful when checking small text files, logs, configuration notes, or command output.

Managing Files

Copying and moving files

Use cp to copy a file. 

hawkode@linux:~$ cp notes.txt backup-notes.txt

Use mv to move or rename a file. 

hawkode@linux:~$ mv notes.txt Projects/notes.txt

The mv command can also rename files. 

hawkode@linux:~$ mv old-name.txt new-name.txt

Deleting Safely

Removing files and folders

Use rm to remove a file. 

hawkode@linux:~$ rm old-notes.txt

Use rmdir to remove an empty folder. 

hawkode@linux:~$ rmdir empty-folder

Be careful with remove commands. Linux usually does not ask twice unless you tell it to. Always check your location with pwd before deleting files.

Permissions

Understanding permissions

File permissions control who can read, write, or execute a file. 

hawkode@linux:~$ ls -l script.sh
-rwxr-xr-x 1 hawkode hawkode 120 Jan 10 12:00 script.sh

In this example:

  • r means read
  • w means write
  • x means execute

Permissions matter in Linux administration, scripting, security testing, and server hardening.

Networking

Useful network commands

The terminal is useful for checking network connectivity and basic troubleshooting. 

hawkode@linux:~$ ping 8.8.8.8
hawkode@linux:~$ ip address
hawkode@linux:~$ ss -tulnp

These commands can help check connectivity, IP information, open ports, and listening services.

Why It Matters

Why this matters in IT and security

The command line is used heavily in networking, Linux administration, cybersecurity labs, log analysis, scripting, and troubleshooting.

Tools like Nmap, OpenVAS, Snort, Metasploit, and many log tools are often used from the terminal. Understanding the basics makes it easier to work through labs, document findings, and troubleshoot issues with more confidence.
