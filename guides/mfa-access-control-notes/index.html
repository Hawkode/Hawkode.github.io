Guide

MFA and Access Control Notes

This guide explains the basics of multi-factor authentication, access control, account security, and common sign-in troubleshooting checks.

The goal is to understand how MFA helps protect accounts, how access decisions are made, and what to check when a user cannot sign in.

Step 1

What is MFA?

MFA stands for multi-factor authentication. It adds an extra verification step when a user signs in.

Instead of using only a password, the user must also prove their identity with another method.

Something you know: password or PIN.

Something you have: phone, authenticator app, security key, or registered device.

Something you are: fingerprint or face recognition.

MFA is important because passwords can be guessed, reused, leaked, or stolen.

Step 2

Understand common MFA methods

Different organisations use different MFA methods. Some methods are stronger than others.

Authenticator app: approval notification or one-time code from an app.

SMS code: code sent by text message.

Phone call: automated call to confirm sign-in.

Security key: physical key used to verify identity.

Windows Hello: PIN, fingerprint, or face sign-in on a trusted device.

Authenticator apps and security keys are usually preferred over SMS because they are stronger options.

Step 3

Understand access control

Access control decides who can access a system, application, file, or service.

In a workplace, access is usually based on user identity, group membership, permissions, device state, and security policies.

User account: the identity used to sign in.

Groups: collections of users used to assign access.

Permissions: what the user is allowed to do.

Roles: admin or service permissions assigned to an account.

Policies: rules that control sign-in and access behaviour.

Step 4

Check the user account status

When a user cannot access a system, first check the account itself.

Is the account enabled? Disabled accounts cannot sign in.

Is the account locked? Too many failed attempts can lock an account.

Has the password expired? The user may need to reset or change it.

Is MFA registered? The user may not have completed MFA setup.

Is the user using the correct username? Check spelling and domain.

These checks are common in Active Directory, Microsoft 365, and Entra ID environments.

Step 5

Check MFA registration

If MFA is required but the user has not registered a method, sign-in may fail or the user may be asked to complete setup.

Registered method: confirm the user has at least one valid MFA method.

Old phone: the user may have changed phone and lost access to the authenticator app.

Wrong number: phone-based MFA may fail if the number is outdated.

Backup method: check whether another sign-in method is available.

In a support role, always follow company process before resetting MFA methods.

Step 6

Check group membership and permissions

Sometimes the user can sign in but cannot access a file, app, mailbox, shared drive, or admin area. This is usually a permissions issue.

Group membership: confirm the user is in the correct security group.

Licence assignment: some cloud apps require the correct licence.

Shared mailbox access: check if the user has been granted access.

Folder permissions: check read, write, modify, or owner permissions.

Role assignment: check whether admin roles are required.

Access should follow least privilege: users should only receive the access they need.

Step 7

Check sign-in conditions

Modern access decisions may depend on more than just the username and password.

Location: sign-ins from unusual countries or networks may be blocked.

Device: unmanaged or non-compliant devices may be restricted.

Risk: suspicious sign-ins may trigger extra checks.

Application: some apps may require stronger sign-in methods.

Browser or session: cached sessions can sometimes cause access issues.

These checks are common in Microsoft Entra ID and conditional access environments.

Step 8

Common user sign-in issues

These are common examples of MFA and access-related problems.

User changed phone: authenticator app is no longer available.

MFA prompt not appearing: notification blocked, phone offline, or app issue.

Access denied: user may not have the correct permission or group membership.

Account locked: repeated failed password attempts.

Licence missing: cloud service access may fail.

Wrong account used: user may be signed into a personal account instead of work account.

Step 9

Basic troubleshooting questions

Ask clear questions before changing account settings.

What application or service are you trying to access?

What error message do you see?

Can you sign in anywhere else?

Did you recently change your password or phone?

Are you using a work account or personal account?

Are you connected from office, home, VPN, or public Wi-Fi?

Good questions help separate password issues, MFA issues, permissions issues, and network or device issues.

Step 10

Document the access issue

Access issues should be documented clearly, especially if they need escalation.

User: affected account or username.

Service: application, mailbox, file share, VPN, or cloud service.

Error: exact message or screenshot if available.

MFA method: authenticator app, SMS, phone call, or security key.

Account checks: enabled, locked, password status, MFA registration.

Permission checks: groups, roles, licences, shared access.

Action taken: what was changed or tested.

Escalation: who it was passed to and why.

Summary

Quick MFA and access checklist

1. Confirm the user, service, and error message.

2. Check if the account is enabled, locked, or expired.

3. Confirm the user has a valid MFA method registered.

4. Check if the user recently changed phone or password.

5. Check group membership, permissions, roles, and licences.

6. Check whether location, device, or policy conditions may block access.

7. Reset MFA only if allowed by company process.

8. Test sign-in again.

9. Document the issue and action taken.

Why It Matters

Why this matters in IT and security

MFA and access control are important parts of account security. They help protect users, company data, cloud services, and internal systems.

Understanding MFA, permissions, and sign-in checks helps with 1st/2nd line support, Microsoft 365 administration, Entra ID, endpoint security, and basic identity protection.
